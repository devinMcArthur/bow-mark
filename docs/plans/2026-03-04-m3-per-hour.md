# Cubic Metres per Hour (m³/h) Secondary Metric Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Show m³/h as a secondary metric (small gray text below T/H) in the Material Productivity table and daily breakdown in the Jobsite Report, and in the Jobsite/Crew Rankings tables on the Dashboard — only for materials/jobsites/crews that had m3-unit shipments.

**Architecture:** Track raw m3 quantity (`SUM(CASE WHEN LOWER(ms.unit) = 'm3' THEN ms.quantity ELSE 0 END)`) alongside tonnes in existing shipment queries. Propagate through the same proportional-allocation pattern used for T/MH. Display only when `totalM3 > 0`. No changes to summary-level stats (overall m³/h cross-material is not meaningful). Follows exact same pattern as the T/MH feature added in the prior commit.

**Tech Stack:** Type-GraphQL (server types), Kysely (PG queries), GraphQL Code Generator (client types), React/Chakra UI (display).
