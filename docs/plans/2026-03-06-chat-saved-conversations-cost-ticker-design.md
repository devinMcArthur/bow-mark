# Chat: Saved Conversations + Per-Conversation Cost Ticker — Design

## Goal

Persist chat conversations per user in MongoDB, allowing users to continue previous
conversations, rename and delete them, and see a real-time cost estimate that accumulates
across the full lifetime of each conversation.

## Data Model

New Mongoose model: `server/src/models/ChatConversation.ts`

```ts
{
  user: ObjectId           // ref to User — owner, from JWT userId
  title: string            // auto-generated by Claude Haiku on first turn; user-editable
  model: string            // e.g. "claude-opus-4-6" — stored for accurate cost calculation
  messages: [              // full conversation history
    { role: "user" | "assistant", content: string }
  ]
  totalInputTokens: number  // cumulative across all sessions, default 0
  totalOutputTokens: number // cumulative across all sessions, default 0
  createdAt: Date
  updatedAt: Date
}
```

Title is auto-generated after the first API turn using a separate `claude-haiku-4-5` call
(cheap, fast) so the title is meaningful rather than a message truncation. Model is stored
per-conversation so cost calculations remain accurate if the model changes later.

## Server Changes

### New REST router: `server/src/router/conversations.ts`

Mounted at `/conversations`. All routes verify JWT and scope to the requesting user's
`userId` decoded from the token.

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/conversations` | List user's conversations — returns `{ id, title, updatedAt, totalInputTokens, totalOutputTokens, model }[]`, no messages |
| `GET` | `/conversations/:id` | Full conversation including messages and token counts |
| `PATCH` | `/conversations/:id/title` | Rename — body `{ title: string }` |
| `DELETE` | `/conversations/:id` | Delete |

All routes return 403 if the conversation's `user` field doesn't match the requesting user.

### Chat router changes: `server/src/router/chat.ts`

**Request body:** add optional `conversationId?: string`

**New SSE event types emitted:**

| Event | Shape | When |
|-------|-------|------|
| `conversation_id` | `{ type: "conversation_id", id: string }` | Immediately after new conversation is created |
| `usage` | `{ type: "usage", inputTokens: number, outputTokens: number }` | After each Claude API turn completes |
| `title` | `{ type: "title", title: string }` | After first turn, once Haiku generates the title |

**Flow:**

1. Decode JWT to extract `userId`
2. If `conversationId` provided: load conversation from DB, verify ownership, use its
   `messages` as history
3. If no `conversationId`: create a new `ChatConversation` document with empty messages,
   emit `conversation_id` SSE event
4. Run the existing agentic loop (unchanged)
5. After each Claude turn: increment `totalInputTokens`/`totalOutputTokens` on the
   document, emit `usage` SSE event
6. Append the new user + assistant messages to the document's `messages` array
7. **First turn only:** fire a non-streaming Haiku call to generate a title, save it,
   emit `title` SSE event

**Title generation prompt (Haiku):**

```
Generate a concise 4-6 word title for a conversation that starts with this message:

"<first user message>"

Respond with only the title. No quotes, no punctuation at the end.
```

## Client Changes

### Layout

The current single centered column becomes a two-column layout:

- **Sidebar (240px, fixed left):** conversation list + "New Chat" button
- **Chat area (fills remaining width):** existing chat UI, unchanged internally

### Sidebar

- "New Chat" button at the top — clears current conversation state
- Scrollable list of conversations sorted by `updatedAt` descending
- Each row: title + relative timestamp (e.g. "2 hours ago")
- Active conversation highlighted
- On hover: inline rename (pencil icon) and delete (trash icon with confirmation popover)
- Rename: click pencil → title becomes an editable input → blur or Enter saves via
  `PATCH /conversations/:id/title`
- Delete: click trash → confirm popover → `DELETE /conversations/:id` → remove from list

### Cost Ticker

Small badge in the chat header, right-aligned next to "Analytics Assistant":

```
~$0.04
```

- Initializes to `0` for new conversations
- When loading a previous conversation: initializes from `totalInputTokens` +
  `totalOutputTokens` using the stored model's rate
- Updates in real-time on each `usage` SSE event

**Pricing constants (client-side):**

```ts
const MODEL_RATES: Record<string, { input: number; output: number }> = {
  "claude-opus-4-6":   { input: 5.00, output: 25.00 },  // $ per million tokens
  "claude-sonnet-4-6": { input: 3.00, output: 15.00 },
  "claude-haiku-4-5":  { input: 1.00, output:  5.00 },
};
```

Cost display: `$${((inputTokens / 1e6) * rate.input + (outputTokens / 1e6) * rate.output).toFixed(4)}`

### New conversation lifecycle (client)

1. User clicks "New Chat" or lands on `/chat` — blank state, no `conversationId`
2. User sends first message → SSE stream starts → `conversation_id` event received →
   store in state
3. `usage` events tick up the cost badge in real-time
4. `title` event received → update sidebar list with generated title
5. User navigates to past conversation → `GET /conversations/:id` → load messages +
   token counts → initialize ticker
